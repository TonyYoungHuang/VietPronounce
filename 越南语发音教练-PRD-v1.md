## 1. 项目摘要

本项目是一款面向中国大陆越南语学习者的微信小程序，核心目标不是讲语法、讲知识点，也不是做 AI 对话，而是聚焦越南语发音纠正。

第一版只上线越南语，并明确区分北越和南越两套发音体系。用户可通过标准示范音、跟读录音、机器评分、错误定位和反复重练来完成系统化发音训练。产品主打零基础到中等水平用户，采用等级制与闯关式纠音路径，核心卖点是“发音纠正更准”。

本产品为独立工具型产品，但商业闭环并不在小程序内直接支付。第一版通过广告投流获取新用户，用户进入小程序后先看卖点页，再选择北越或南越，体验少量免费试练内容；体验后由结果页引导联系客服，由客服承接成交，用户在现有电商平台购买后获得兑换码，再返回小程序完成开通。

## 2. Problem Statement

### 2.1 用户问题

中国大陆的越南语学习者，尤其是零基础、初级和中等水平用户，普遍面临以下问题：

1. 知道要开口练，但缺少一个专门做越南语发音纠正的工具。
2. 普通课程更偏知识输入，无法持续告诉用户“这句到底读得准不准”。
3. 很多语言产品把越南语当作单一标准，不区分北越与南越，容易导致标准音和纠音结果混乱。
4. 用户需要反复练习，但如果只有一个总分，没有具体错位提示，练习无法持续。
5. 用户更需要“纠音训练器”，而不是开放式对话、真人老师或复杂课程平台。

### 2.2 商业问题

1. 现有越南语学习市场中，缺少一个强卖点明确的“发音纠正”型产品。
2. 广告投流需要一个强体验型入口，能够让用户在短时间内感知到价值并愿意付费。
3. 需要验证“免费试练 -> 客服承接 -> 电商购买 -> 兑换码开通”的闭环是否成立。

### 2.3 核心判断

本产品第一版要解决的不是“如何让用户学会全部越南语”，而是：

> 如何让中国大陆学习者以较低门槛进入越南语发音训练，并在明确标准、明确错误、可反复重练的机制下，感知到产品的纠音价值，从而形成付费转化和持续练习。

## 3. Proposed Solution

### 3.1 产品定义

提供一款只做越南语的微信小程序，定位为“发音教练”，通过固定内容的跟读评分完成机器纠音训练。

### 3.2 核心方案

1. 首发只上线越南语。
2. 越南语必须区分北越和南越两套发音体系。
3. 用户第一次正式进入学习路径前，先选择北越或南越。
4. 用户购买后，北越和南越两个体系都解锁，但两套体系的进度、成绩和报告完全独立。
5. 第一版以等级制组织内容：零基础、初级、中等。
6. 训练路径按“音素/单音 -> 音节 -> 词 -> 句子”逐级推进。
7. 每条内容都提供标准示范音、原文、中文释义、中文近似发音提示、录音入口、评分结果。
8. 每次录音后一次性返回固定结构结果：总分、完整度、准确度、流利度、错误定位、是否达标。
9. 错误定位至少精确到音节 / 声调级。
10. 评分重点偏准确度，突出“纠音更准”的卖点。
11. 用户可回放自己的录音，并与标准音对比，支持无限重练。
12. 练习机制采用统一达标线；未达标项允许跳过查看后续内容，但不计入正式进度。
13. 已通过内容可随时回看回练。

### 3.3 第一版内容策略

第一版采用“小而精”策略，不追求一次性铺满全部等级内容。

首发范围：
- 零基础完整
- 少量初级句子

目标是优先验证：
- 用户是否认可“机器纠音”价值
- 产品是否能驱动反复练习
- 免费试练是否能带来有效转化

### 3.4 商业路径

1. 广告投流获取新用户。
2. 用户进入小程序后先看卖点页。
3. 用户先选择北越或南越。
4. 用户进入免费试练。
5. 用户完成试练后看到完整纠音结果。
6. 结果页明确引导联系客服。
7. 客服负责成交解释与购买引导。
8. 用户在现有电商平台完成购买。
9. 电商平台发放兑换码。
10. 用户回到小程序输入兑换码完成开通。

## 6. Success Criteria

### 6.1 业务指标

- 免费试练 -> 付费购买转化率 >= 10%

### 6.2 留存指标

- 次日留存 >= 30%

### 6.3 产品体验指标

满足以下定性目标：
- 用户能顺畅完成“听标准音 -> 跟读 -> 出分 -> 看错误 -> 重练”的完整闭环。
- 用户能明确感知产品核心卖点是“发音纠正更准”。
- 用户能理解北越和南越两套体系是独立标准。
- 用户能看懂错误定位，并据此进行重练。
- 用户愿意复练而不是只看分数离开。

### 6.4 MVP 成立标准

- 仅依靠“零基础完整 + 少量初级句子”，产品也能形成可用闭环。
- 不依赖真人服务、不依赖自研评分模型，也能完成第一轮市场验证。

## 7. 技术约束

### 7.1 平台约束

- 第一版只做微信小程序。
- 界面语言以简体中文为主。
- 架构需为未来接入其他东南亚语言预留扩展空间。

### 7.2 内容与评分约束

- 第一版只上线越南语。
- 标准音必须区分北越和南越。
- 免费试练内容也必须区分北越和南越。
- 评分结果必须为固定结构结果。
- 错误定位至少到音节 / 声调级。
- 单条内容以音、音节、词、短句为主。
- 第一版采用统一固定达标线。

### 7.3 能力实现约束

- 第一版评分引擎优先接入成熟第三方能力。
- 标准示范音频优先采用高质量 AI 语音。
- 产品必须联网使用。
- 评分返回目标为 3 秒内。

### 7.4 账号与数据约束

- 免费试练可免登录。
- 正式学习、保存进度、兑换码开通必须登录。
- 登录方式为微信登录 + 绑定手机号。
- 每条练习至少保留最近一次成绩与是否达标。
- 原始录音仅短期保存，用于本次评分和近期展示，后续清理。

### 7.5 开通与运营约束

- 一个兑换码只能使用一次，并绑定一个账号。
- 已开通用户再次输入兑换码时，提示“已开通，无需重复兑换”。
- 第一版先用内部简易方式维护内容和兑换码，不强制做正式后台。
- 第一版不强制做投放埋点和漏斗分析系统。

## 8. 用户分层与典型场景

### 8.1 用户分层

1. 零基础用户
   - 目标：先读准音、调和基础词句
   - 需求：清晰标准音、拼读提示、低门槛开口

2. 初级用户
   - 目标：修正已经形成的错误发音
   - 需求：更稳定的评分和错误定位

3. 中等水平用户
   - 目标：保持标准度并优化句子发音
   - 需求：更高质量的句子级训练与复练

### 8.2 关键场景

1. 投流用户首次进入
   - 想快速知道产品是否真的能纠音

2. 免费试练后犹豫购买
   - 需要看到明确结果，并被顺畅承接到客服

3. 已开通用户持续学习
   - 想快速返回上次练到的地方，继续纠音闯关

4. 用户切换北越 / 南越
   - 想比较两套体系，但不希望进度混在一起

5. 用户反复不过关
   - 希望可以无限重练，并随时听标准音和自己的录音

## 13. MVP 开发建议节奏

### 阶段 1：跑通核心闭环

目标：先跑通“体验 -> 购买 -> 开通 -> 学习”的最小链路。

建议范围：
- 卖点页
- 体系选择
- 免费试练
- 试练结果页
- 联系客服
- 登录 / 绑手机
- 兑换码开通
- 首页
- 单条练习页
- 结果页

### 阶段 2：补齐正式学习路径

目标：形成可以持续使用的纠音训练器。

建议范围：
- 等级页
- 章节页
- 最近一次成绩保存
- 已通过内容回练
- 北越 / 南越独立进度

- 简易兑换码维护方式

## 15. 核心交互规则

### 15.1 达标规则

- 所有条目统一固定达标线。
- 达标后计入正式进度。
- 未达标时可重练。
- 未达标时可跳看后续内容，但不计入正式进度。

### 15.2 重练规则

- 支持无限次重练。
- 用户可随时回听标准音。
- 用户可回放自己的录音。

### 15.3 进度规则

- 北越和南越进度完全独立。
- 已通过内容可随时回看回练。
- 每条条目至少保存最近一次结果。

### 15.4 账号规则

- 免费试练可免登录。
- 正式学习必须登录并绑定手机号。
- 兑换码开通必须在已登录状态完成。

## 16. PRD 结论

第一版不追求成为一个大而全的语言学习平台，而是要把“越南语机器纠音训练器”这个最核心价值打透。

只要第一版能够做到以下几点，就具备成立基础：
- 让用户明显感觉评分和纠音有价值
- 让用户能明确知道错在哪里
- 让用户愿意继续重练
- 让免费试练顺畅导向客服与购买
- 让购买、兑换、开通和继续学习形成闭环

换句话说，第一版的目标不是“内容最多”，而是“纠音价值最清楚，转化路径最成立”。
