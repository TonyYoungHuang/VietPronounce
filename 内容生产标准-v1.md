# 越南语内容生产标准 v1

本标准用于约束越南语纠音小程序的课程内容，目标是：

- 以后补内容时不乱
- 后台批量维护时不崩
- 北越 / 南越可以镜像扩容
- 后续扩到其他语种时也有复用模板

## 1. 单条练习字段标准

每条练习必须统一使用以下字段：

- `id`
- `type`
- `text`
- `translation`
- `hint`
- `demoAudio`
- `segments`

### 字段说明

- `id`：全局唯一，建议带上方言、等级、lesson 骨架
- `type`：只允许 `syllable` / `word` / `sentence`
- `text`：越南语原文
- `translation`：中文释义
- `hint`：中文近似提示，仅做入门辅助
- `demoAudio`：示范音频路径
- `segments`：机器纠错分段数组，至少 1 项

### `segments` 子项标准

每个 `segment` 必须包含：

- `text`
- `tip`

示例：

```json
{
  "id": "north-beginner-greeting-1-item-1",
  "type": "syllable",
  "text": "xin",
  "translation": "问候开头音节",
  "hint": "近似：辛",
  "demoAudio": "/assets/audio/north-demo.wav",
  "segments": [
    {
      "text": "xin",
      "tip": "x 的摩擦感要更细，不要读成普通的“辛”。"
    }
  ]
}
```

## 2. `type` 固定分层

系统只允许 3 个内容类型：

- `syllable`：音节
- `word`：单词 / 固定词
- `sentence`：句子 / 场景表达

不新增第四种类型。  
如果后续需要“连续表达”，仍归入 `sentence`，通过句长和内容场景区分。

## 3. 各等级内容配比

### 零基础 `beginner`

定位：

- 音节
- 单词
- 极短句

建议配比：

- `syllable`：35%~45%
- `word`：35%~45%
- `sentence`：10%~30%

最低要求：

- 必须同时包含 `syllable` / `word` / `sentence`

### 初级 `elementary`

定位：

- 高频短句
- 固定表达

建议配比：

- `word`：20%~40%
- `sentence`：60%~80%

最低要求：

- 必须包含 `sentence`
- 允许保留少量 `word`
- 不再新增 `syllable`

### 中等 `intermediate`

定位：

- 场景句
- 连续表达

建议配比：

- `sentence`：100%

最低要求：

- 只允许 `sentence`

## 4. 北越 / 南越镜像结构规则

为了后续批量扩内容，北越与南越必须遵守镜像结构：

- 等级顺序完全一致：`beginner -> elementary -> intermediate`
- 同一等级下，课时数量一致
- 镜像课时的 `lesson id` 去掉 `north-` / `south-` 前缀后必须一致
- 镜像课时的 `item` 数量一致
- 镜像课时的 `item type` 序列一致
- 镜像 `item id` 去掉 `north-` / `south-` 前缀后必须一致

### 镜像示例

- 北越：`north-beginner-greeting-1`
- 南越：`south-beginner-greeting-1`

它们必须被视为一对镜像课时。

## 5. ID 命名建议

### lesson id

建议格式：

```text
<dialect>-<level>-<lessonKey>
```

例如：

- `north-beginner-greeting-1`
- `south-beginner-greeting-1`

### item id

建议格式：

```text
<dialect>-<lessonKey>-item-<n>
```

例如：

- `north-beginner-greeting-1-item-1`
- `south-beginner-greeting-1-item-1`

## 6. 内容生产建议流程

推荐以后按以下顺序补内容：

1. 先定某个等级的 lesson 骨架
2. 先做北越 / 南越的镜像 lesson 列表
3. 再按镜像 lesson 填 item
4. 最后分别补两边的发音差异说明和 `tip`

这样可以保证：

- 结构一致
- 后台好维护
- 后续好做批量导入
